Accessibility Notes

Regardless of any digital rights management or the license terms a digital book is released under, libraries and educational facilities and probably some other institutions have the legal right—and perhaps legal obligation in some cases—to modify digital books to reduce or eliminate accessibility barriers for their users that need them mitigated. Legalities aside, when a digital publication can be modified to mitigate or reduce an accessibility barrier it certainly is the moral thing to do.

To help facilitate this, every page in the ePub spine references the file css/a11y.css as the last style sheet that is referenced so that any accessibility fixes that can be made with CSS can be placed in that file and they will always override any default styles defined by the ePub viewer or any CSS definitions defined in this ePub that may be causing accessibility issues for a user. Modification of that file does require some technical skill.

It is the explicit goal of Pipfrosch Press for everything we publish to meet the WCAG 2.1 ‘AA’ conformance level and it is the desire of Pipfrosch Press to meet as many ‘AAA’ success criteria as practical within the current funding limitations.

Not all accessibility barriers have corresponding WCAG 2.1 success criterion. When aware of such barriers, Pipfrosch Press generally considers them to be ‘AAA’ issues which means we try to provide solutions when practical to do so within the current funding limitations but may not always be able to.

Visual Technical Nomenclature Terms

Some technical terms used in this ePub—such as mammalian dentition notation—has a visual aspect to the nomenclature.

Assistive technology for the print disabled may not present those terms to users in an optimal way without proper tagging instructing the assistive technology in how to present the terms to the user. With mathematical equations, MathSpeak can be used to solve this issue but I am currently ignorant on what the proper methods are for visual presentations of terms that are not mathematics.

I was not able to find the specific WCAG success criterion this violates but Pipfrosch Press considers this to be a level ‘AA’ conformance issue. This ePub is intended to convey scientific information which makes proper markup of technical terms that have a visual component critical. This must be solved before the initial official release of this ePub.

Contrast Issue

The covers for the original ‘Numbers’ in Volume 1 had a greenish background color behind the artwork and text. For nostalgia reasons I have attempted to emulate those covers including a greenish background color.

The background color I chose has a 7.73:1 contrast ratio with the ‘not quite but nearly black’ text color and does pass the necessary contrast threshold for WCAG ‘AA’ Success Criterion 1.4.3 at all font sizes but it barely passes ‘AAA’ Success Criterion 1.4.6 and may present issues for the text that uses a smaller than ‘normal’ font size. The font size used on the emulated covers is actually determined by the display screen size, so that may happen.

Contrast Issue Solution

For a user who needs more contrast than the greenish emulated covers provide, add the following to the css/a11y.css file:

.svgcoverbg { fill: none; }

That will remove the purely nostalgic background color from the emulated covers.

Italicized Text Issue

For some users, including but not exclusively many dyslexic users, italicized text can be an access barrier. I am not able to find a specific WCAG success criterion related to italicized text but that does not make this issue any less real.

A web page that does discuss dyslexic accessibility issues: 6 Surprising Bad Practices That Hurt Dyslexic Users.

Italicized Text Solution

To accomodate users who have issues with italicized text, there are already CSS definitions in a file called css/noitalics.css to override most of the uses of italicized text within this ePub but in a visually meaningful way so that the visual cues are not lost.

Copy the contents of that file into the css/a11y.css file and most of the italicized text will be replaced with upright text, with the following exceptions:

  1. The ePub cover is a bitmap image, text on the ePub cover can not be altered with CSS. This is a limitation of ePub and digital books in general. The cover is primarily intended for the digital library and ePub viewers need to turn it into a thumbnail sized for the library view without being expected to rasterize a vector image.
  2. A few technical nomenclature terms—such as the dental notation pm2—remain italicized.

In the second case, if you really need to override the italics you still can by also adding the following to the css/a11y.css file:

.nomen { font-style: normal; } span.lingual { font-style: normal; }

Figure Font Size

I currently classify this as a ‘AAA’ issue although it may actually be a ‘AA’ issue (Success Criterion 1.4.4 depending upon if the text is considered a caption). I am not able to find a specific ‘AAA’ success criterion for this issue but that does not make it any less real.

When a user needs a larger font size, increasing the font size used by the ePub viewer will not increase the font size used in figures that are drawn with SVG. There is not an easy solution to this problem that I am aware of.

Even though it should be technically possible for an ePub viewer to allow the user to zoom in on a part of a figure as needed, it seems that very few ePub viewers actually implement this kind of functionallity. I have personally only seen it implemented when an SVG image is embedded in an ePub page using the <img/> tag, but that is not how figures are done in this ePub as that method does not seem to allow the image to have access to fonts embedded within the ePub.

I am hopeful that in the near future I will either discover an existing solution to this issue or if one does not yet exist, that a solution will be developed that ePub viewers implement.

Phonetic Markup

This is ‘AAA’ Success Criterion 3.1.6.

The ePub 3.2 specification specifically provides a mechanism for adding phonetic markup to words and phrases that a text to speech synthesiser (TTS) is likely to mispronounce or for which pronunciation may be ambiguous based on the spelling. This ePub is currently woefully inadequate in implementing such markup.

Presently I am not aware of any ePub viewers that actually implement this part of the ePub 3.2 standard but I still want to provide the proper phonetic markup so that if and when a viewer does support it, this ePub will already be properly tagged.

‘Æ’ and ‘Œ’ Ligatures

Journal of Mammalogy Volume 1 made heavy use of the ‘æ’ ligature and some use of the ‘œ’ ligature. These ligatures can cause the same issues for some users that italicized text causes.

It is the opinion of Pipfrosch Press that ePub viewers should have a setting that automatically expands those ligatures but Pipfrosch Press is also aware that having an opinion does not actually mitigate the resulting access barrier for impacted users.

This is not something can be solved with a CSS declaration. A version of this ePub with those ligatures expanded—along with the previously mentioned css/noitalics.css style sheet already applied—will be available from the Pipfrosch Press website.

High Contrast Text

For some users, including many dyslexic users but also others, too much contrast between the text and the background can be an accessibility barrier that increases difficulty in reading and reduces comprehension.

Pipfrosch Press does not define a text color or background color for the main content, doing so would interfere with the operation of ‘night mode’ with some ePub viewers. However it is relatively easy to set.

High Contrast Text Text Solution

The ePub viewer really should allow the user to define the text color and background color but in the event the ePub viewer a users uses does not and the contrast is too high, add the following to the css/a11y.css file:

body { color: #222222; background-color: #ededed; }

That combination of nearly but not quite black on top of a light gray background works very for me, but of course you can adjust the colors as needed.
